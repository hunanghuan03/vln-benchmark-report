VLN Benchmark 指令风格调研

围绕 7 个 Vision-and-Language Navigation 工作的指令"问法"、GT 轨迹设计、自动生成方法的横向对比与选型建议。

为什么要做这个调研?
我们要发的是新 benchmark 论文,而不是方法论文。要让现有 SOTA 模型能在我们的 benchmark 上"涨点",生成的 instruction 必须与主流风格(尤其 R2R 系列)对齐。本调研梳理 7 个代表性工作的指令风格、轨迹设计与生成方法,作为我们 instruction generation 模块的设计依据。

核心结论

✓
对齐 R2R 风格是正确决定。几乎所有近年 SOTA(BEVBert, ETPNav, ScaleVLN, NaviLLM, DUET, VLN-BERT…)的训练范式、loss、agent 架构、tokenizer 词表都是为 短中程 R2R-style 单段指令设计。把 R2R 模型直接放到 LH-VLN 上实测 SR = 0%,完全跑不动 — 兼容性验证落实在数据上。
★
最值得参考的工作: ① HA-R2R(LLM 自动改写 R2R-style,prompt 直接可复用) ② VLN-CE(连续空间动作设计与 episode JSON schema) ③ RoomTour3D(多模型 caption pipeline) ④ C-Instructor(CoTL 思想 — 先抽 landmark 再生成)。

风格直观对比

每种风格用一句中文片段感受差异:

R2R · 短、祈使句、地标点名
所有 SOTA 模型的训练目标。我们要复制的"模板"。
"上楼,经过钢琴,在鹿角旁等待。"
RxR · 第二人称、密集地标 + 状态确认
长且口语化,适合做 fine-grained 对齐。
"现在你正站在一扇关着的门前,向左转,你可以看到两级木台阶……"
HA-R2R · R2R 风格 + 人体活动 + 社会感知
在 R2R 基础上叠加,保留兼容性。
"继续向前走,确保不要打扰他的寻找……注意不要撞到他们。"
RoomTour3D · 物体进展叙述 + R2R 动作骨架
来自房产视频的"摄影解说"风格。
"穿过走廊向前走,经过墙上的一幅画,从远处床、窗帘和台灯的视角开始……"
LH-VLN · 高层目标、多阶段、"……,然后……"
语气与 R2R 完全不同,这是为什么我们不对齐它。
"把浴室里的毛巾拿到厨房中岛,然后抓起茶壶放到客厅茶几上。"

给我们项目的建议

指令风格层面

  1. 主对齐 R2R:25–35 词、纯祈使句、以 stop/wait 收尾、每句一个具体地标。
  2. Prompt 直接复用 HA-R2R 的 system + few-shot 模板(详见"推荐 Prompt 方案"页)。
  3. 引入 C-Instructor 的 CoTL 思想:先用 LLM 抽 landmark list,再 conditioned 生成 R2R-style 指令。我们 3DGS 已有 bbox,等于自动获得 ground-truth landmark。

GT 轨迹层面

  1. 动作空间直接抄 VLN-CE:FORWARD 0.25m + TURN ±15° + STOP;成功半径 3.0m。
  2. 采样规则参考 R2R:不同 region 之间的 start/goal,navmesh 最短路径,过滤 <5m,4–6 段 edges(转化为"经过 4–6 个房间")。
  3. Episode JSON schema 用 VLN-CE 的:Habitat 兼容,所有 SOTA 都能加载。

风格特点总结

平均长度 29 词(论文明确数字)
词表大小 ~3.1k 词;~1.2k 高频词
句式 纯祈使句、无人称主语
指令粒度 单段(隐含 3-5 sub-instruction)

典型动词模式

walk past / through / down、turn left/right、go up/down、pass the X、exit / enter、stop at / wait by / wait next to、head + 方向

地标描述方式

明确点名物体(piano, archway, moose antlers, bathtub, double glass doors, wine bottles, office 359),通常作为 turning point 或 stopping point。每句必含一个具体地标作参照。

GT 轨迹设计

场景来源 Matterport3D (MP3D)
场景数量 90 (61 train / 11 val unseen / 18 test)
轨迹数 7,189 paths · 21,567 instructions
平均路径 10m, 4-6 edges
起点/终点策略 不同 room 自动采样 + 最短路径
动作空间 离散 nav-graph(平均 degree 4.1)

JSON 数据格式(官方 R2R README)

{
  "distance": float,           // path 距离 (米)
  "scan": str,                 // MP3D 场景 ID
  "path_id": int,              // 路径唯一 ID
  "path": [str, str, ...],     // viewpoint ID 序列
  "heading": float,            // 起始 heading (radians)
  "instructions": [str, str, str]   // 3 条 human instructions
}

如何生成 instruction

i
R2R 是 100% 人工标注,使用 Amazon Mechanical Turk(AMT)。无 LLM 参与。我们项目无法照搬,但需要它的"风格"。

人工标注流程

1workers 在 WebGL 3D viewer 中以 fly-through 方式浏览路径
↓
2可在任意 viewpoint pan / tilt 相机查看上下文
↓
3写下"指挥智能机器人到达终点"的指令
↓
4每条路径采集 3 条独立的 instructions

关键参数

  • 400+ US-based AMT workers
  • ~1,600 总标注小时
  • 预筛选 worker 历史质量
  • 无自动验证机制,仅靠每路径 3 条提供多样性
论文中关键引用:"…22k instructions are available, with an average length of 29 words."

风格特点总结

平均长度 78 词(en-US 子集 129 词)
规模 126K instructions × 3 语种
句式 第二人称(you)+ 大量祈使
地标密度 每条 6-8 次 entity reference(R2R 是 3.7)

独特特征

  • State verification:出现"you are in the balcony area facing towards…"这种状态确认句
  • Allocentric relations 高:物体之间相对关系密集(20% → 68-92%)
  • Coreference 多:24-76% 的句子有指代回指
  • Timestamp + pose 对齐:每个词都有起始与结束时间戳,与 6-DoF camera pose 同步

GT 轨迹设计

场景来源 同 R2R(MP3D 90 scenes)
轨迹数 16,522 paths(R2R 的 2.3 倍)
平均长度 14.9m, 8 edges
非最短路径 44.5%(刻意打破 R2R 偏置)

两级路径采样(创新点)

1Room graph 层:采样 simple room paths(≤5 rooms, ≤2 floors)
↓
2Panorama 层:在子图内跑最短路径
↓
3Greedy selection:保证 panorama 覆盖均匀,反偏置

四大设计目标(论文 path desiderata):①路径长度高方差 ②允许间接接近终点 ③自然性(不绕圈) ④均匀环境覆盖。

如何生成 instruction

i
RxR 是 100% 人工标注 + 语音录制 + 自转写,但有完善的双重验证机制(Guide-Follower)。我们的项目可以借鉴它的"质量验证流水线"思想。

Guide-Follower 双任务流程

1Guide 任务:annotator 看路径,说出导航指令(不是写),tool 录音 + 同步记录 6-DoF camera pose
↓
2自转写:Guide 自己转写自己的录音 → 高质量文本
↓
3对齐:ASR + dynamic time warping → 每词与 pose 时间对齐
↓
4Follower 任务:另一 annotator 听音频走路径;若失败,Guide 重做
↓
5Human follower 成功率 93.9% → 数据质量验证
Guide 任务平均 458 秒/条,Follower 平均 132 秒/条。三种语言:英、印地、泰卢固(各 14K paths × 3 instr)。
经典基准 · 03

VLN-CE · Vision-and-Language Navigation in Continuous Environments

把 R2R/RxR 从离散 nav-graph 移植到 Habitat 连续 mesh。我们 3DGS 项目轨迹设计的直接参考。

ECCV 2020 Krantz et al. 轨迹参考

指令实例

i
VLN-CE 不创建新的 instruction 语料,而是复用 R2R 和 RxR 的指令文本。所以指令风格、长度、句式完全等同于 R2R 或 RxR。
→ 样例请参考 R2R 页面(R2R-CE)或 RxR 页面(RxR-CE)。

真正的贡献:连续空间

VLN-CE 论文真正的贡献在于把 nav-graph 离散任务变成连续 3D mesh 任务,这正是我们 3DGS 项目要做的事。

场景表示 Habitat 连续 mesh(基于 MP3D)
场景数量 90(同 R2R)
动作空间 低层离散动作集
成功半径 3.0m(v1-3 标准)

动作空间(★ 核心可复用)

  • MOVE-FORWARD 0.25m — 前进 0.25 米
  • TURN-LEFT 15° — 左转 15 度
  • TURN-RIGHT 15° — 右转 15 度
  • STOP — 停止(必须由 agent 主动选择)

专家轨迹动作分布(论文原文):68% forward, 15% turn-left, 15% turn-right, 2% stop。

GT 轨迹设计

从 nav-graph 移植的过程

1读取 R2R 原始 nav-graph 的 viewpoint 序列
↓
2把每个 viewpoint 的 (x, y, z) 坐标对应到 Habitat mesh
↓
3用 navmesh shortest-path 连接相邻 viewpoint,得到连续 reference path
↓
4过滤 mesh 重建有缺陷、navmesh 不可达的 episode

JSON Episode 格式(★ 我们项目应直接复用)

{
  "episode_id": int,
  "scene_id": str,           // .glb 路径
  "trajectory_id": int,
  "start_position": [x, y, z],
  "start_rotation": [qx, qy, qz, qw],
  "instruction": {
    "instruction_text": str,
    "instruction_tokens": [int, ...]
  },
  "goals": [{"position": [x, y, z], "radius": 3.0}],
  "reference_path": [[x, y, z], ...],   // 连续 waypoints
  "info": {"geodesic_distance": float}
}

如何生成 instruction

i
VLN-CE 的 instructions 完全继承自 R2R / RxR 的人工标注,论文不专门生成新语料。可借鉴的是它的 episode 格式与动作空间。

给我们项目的关键启发

✓
所有数字直接照搬到 3DGS 项目:
  • 动作步长:0.25m / 15°(标准)
  • 成功半径:3.0m
  • 路径长度过滤:> 5m
  • Episode JSON 用上面这套 schema

如何生成 instruction(★ 核心)

System Prompt(完整原文 + 翻译)

英文原文
You are tasked with generating an objective and detailed description of a path and the related scene information based on the given input. Focus solely on observable entities and actions in the scene, avoiding descriptions of smell, atmosphere, or subjective feelings. Ensure clarity and precision in object and human activity descriptions.
中文翻译
你的任务是基于给定的输入,生成一段关于路径以及相关场景信息的客观、详细的描述。只关注场景中可观察到的实体和动作,避免描述气味、氛围或主观感受。确保物体和人类活动描述的清晰性和精确性。

User Prompt(完整原文 + 翻译)

Input:
[instruction_text]: "..."
[start]: "..."
[during]: "..."

如何生成 instruction(★ Pipeline 完整解析)

这是一个多模型协作的全自动 pipeline。我们 3DGS 项目可以借鉴它的工程结构。

完整流水线

1视频采样 · 每 2 秒一帧(对应 1.42 m/s 人类步速)
↓
2BLIP-2 · VQA "Which room am I in?",16 选 1(bedroom/kitchen/...)。50-clip 验证准确率 85%
↓
3RAM · 给每帧打开放词表 object tags
↓
4Grounding-DINO · 给每个 object 定位 bbox
↓
5Depth-Anything · 估计相对深度
↓
6模板化 caption · "There is a <obj> to the <direction> in <distance>"
↓
7GPT-4-Turbo · 任务指令 + 2 in-context examples + frame captions → 最终 instruction

关键模板(中间步骤,可直接抄)

"There is a <object tag> to the <spatial position> of current spot in <relative distance>."

# 距离三档:near distance / closer distance / further distance
# 方位三档:left / middle / right

关键点:不写绝对坐标,只写相对位置。这点对我们 3DGS 项目很有借鉴价值。

GPT-4 Task Instruction(完整原文 + 翻译)

英文原文
You will be given a set of continuous frames. The frames are captured during the camera movement. During movement, the objects in the frames change gradually, like objects passing by, objects moving towards somewhere.

You should return a single and concrete sentence describing the camera moving trajectory by the object's progression in the frames. You don't need to mention all the objects. It is good to describe the moving trajectory without all of the objects.
中文翻译
你将得到一组连续的视频帧。这些帧是在相机移动过程中拍摄的。在移动过程中,帧中的物体逐渐变化,比如物体被经过、物体朝某处移动。

你应该返回一句单独的、具体的句子,通过帧中物体的渐变过程来描述相机的移动轨迹。你不需要提及所有物体。在不提及所有物体的情况下描述移动轨迹反而是好的。
★
这两条约束最值钱: ① "single and concrete sentence" — 否则 GPT-4 会写小作文;② "you don't need to mention all the objects" — 让 GPT-4 学会取舍。

C-Instructor

基于 LLaMA-7B 训练的可控 instruction generator。CoTL 思想:先抽 landmark,再生成完整指令。

ECCV 2024 Kong et al. 思想可借鉴 arXiv:2407.07433

指令实例

C-Instructor 是生成器模型,不是新数据集。指令风格随训练目标(R2R / REVERIE / R2R-Last)切换。

样例 1 · 成功生成(R2R 风格,Figure 7)

英文原文(C-Instructor 生成)

中文翻译
走出卧室,然后穿过走廊……

样例 2 · 失败案例对照(Figure 8)

英文原文 · 人工标注 (Annotation)
Turn right and exit the bedroom. Turn left and walk through the doorway. Wait at the top of the stairs.
中文翻译
向右转,走出卧室。向左转,穿过门口。在楼梯顶端等待。

Exit the bedroom and walk through the hallway…

方法特点总结

基础模型 LLaMA-7B + LLaMA-Adapter
视觉编码 CLIP-ViT-L-14 + 8 ViT blocks
训练范围 仅 finetune LLM 最后 2 层
数据 复用 R2R / REVERIE / PREVALENT

三个核心组件

① CoTL · Chain of Thought with Landmarks(★ 最值得借鉴)

英文原文
CoTL employs a step-by-step thinking approach to guide the model to identify crucial landmarks before generating complete instructions.
中文翻译
CoTL 采用分步思考的方法,引导模型在生成完整指令之前先识别关键地标。

Landmark 来源:

  • Λ_x — 从训练集 instruction 文本里抽出的 landmark
  • Λ_a — 从视觉 panorama 用 spatial-temporal 选取的 landmark(score threshold β=0.25)

② STMT · Spatial Topology Modeling Task

英文原文
STMT incorporates spatial connectivity prediction as an auxiliary task in training to facilitate the understanding of the topological structure of the environment. ... we make the model predict how to return to the previous location from the current viewpoint.
中文翻译
STMT 将空间连通性预测作为训练中的辅助任务,以促进对环境拓扑结构的理解。……我们让模型预测如何从当前视角回到上一个位置。

这是辅助 loss,作用是让 trajectory encoder 学到更好的空间表征。

③ SMT · Style-Mixed Training

英文原文
Distinct instruction styles are trained using prompts as differentiation, enabling control over the style of generated instructions.
中文翻译
不同的指令风格使用 prompt 作为区分进行训练,从而能够控制生成指令的风格。

实操层面:同一模型,不同 prompt prefix:

  • "Generate an R2R-style instruction:" → R2R 风格
  • "Generate a REVERIE-style instruction:" → REVERIE 风格(短、目标对象式)
  • "Generate an R2R-Last-style instruction:" → 只描述最后一段

给我们项目的启发(★ 思想降级版)

✓
虽然我们大概率不会自己训生成器(成本高),但把 CoTL 思想搬到 prompt 里是免费的:把"先抽 landmark 再生成"做成两步 prompt。我们 3DGS 已有 bbox 标注 → 直接当 ground-truth landmark 喂给 GPT-4。 
第 1 步 Prompt:
"Given the trajectory and bounding boxes,
list the key landmarks the agent will pass."

第 2 步 Prompt:
"Based on the landmarks above,
generate an R2R-style instruction in 25-35 words."

评估结果(论文 Table 1)

在 R2R val unseen 上,SPICE 比之前最佳提升 3.8 (绝对) / 21.8% (相对)。这证明 landmark-first 策略真的有效。
